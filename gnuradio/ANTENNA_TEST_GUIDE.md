# USRP天线对天线测试指南

## 硬件连接

### 天线连接方式
```
USRP B210
├── TX/RX 端口 ──[天线1]── (空中传输) ──[天线2]── RX2 端口
```

### 天线摆放建议
1. **距离**: 10-50cm（根据信号强度调整）
2. **方向**: 两个天线平行或相对
3. **环境**: 尽量避开金属物体和其他无线设备

### 参数调整
由于没有衰减器，已调整参数：
- **TX增益**: 10 dB（降低发送功率，避免接收端过载）
- **RX增益**: 40 dB（提高接收灵敏度）

## 参数调优

如果接收信号太强或太弱，可以调整：

### 方法1: 调整天线距离
- **信号太强**: 增加天线距离
- **信号太弱**: 减小天线距离

### 方法2: 调整增益参数
编辑 `usrp_loopback_test.py`：

```python
# 如果接收信号太强（过载）
TX_GAIN = 5   # 降低发送功率
RX_GAIN = 35  # 降低接收增益

# 如果接收信号太弱
TX_GAIN = 15  # 提高发送功率
RX_GAIN = 45  # 提高接收增益（最大50）
```

### 方法3: 使用命令行参数
```bash
# 直接运行Python脚本并观察信号强度
python3 usrp_loopback_test.py \
    --serial "serial=7MFTKFU" \
    --model ../CCNN/2_models/best/ccnn_epoch_86_acc_0.9913.pth \
    --type tone \
    --tests 5
```

## 信号质量检查

### 使用signal_receiver.py查看实时信号
```bash
# 启动实时监控（会显示图形界面）
python3 signal_receiver.py \
    --serial "serial=7MFTKFU" \
    --tx-type tone \
    --tx-gain 10 \
    --tone-freq 500e3
```

观察：
- **时域信号**: 幅度应该稳定，不应该削波
- **频谱**: 应该能看到明显的峰值
- **功率**: 应该在-60dB到-20dB之间

## 常见问题

### Q1: 接收不到信号
**解决方法**:
1. 检查天线是否正确连接
2. 减小天线距离（试试5-10cm）
3. 提高TX增益到15-20 dB
4. 检查天线方向

### Q2: 信号过载（Overflow）
**解决方法**:
1. 增加天线距离
2. 降低TX增益到5 dB
3. 降低RX增益到30 dB

### Q3: 检测准确率很低
**可能原因**:
1. 信号质量差（SNR太低）
2. 环境干扰严重
3. 天线位置不佳

**解决方法**:
1. 调整天线距离和方向
2. 换一个电磁环境更干净的地方
3. 增加测试次数观察稳定性

### Q4: 环境干扰
**识别方法**:
```bash
# 只接收不发送，看是否有信号
python3 signal_receiver.py --serial "serial=7MFTKFU"
```

如果没有发送也能看到强信号，说明有环境干扰。

**解决方法**:
1. 换一个频率（修改CENTER_FREQ）
2. 换一个测试地点
3. 关闭附近的WiFi/蓝牙设备

## 预期性能

### 天线测试 vs 同轴线测试
- **同轴线+衰减器**: 准确率通常 > 85%
- **天线对天线**: 准确率可能 70-85%（受环境影响）

天线测试的准确率可能略低，因为：
1. 信号路径不稳定
2. 多径效应
3. 环境干扰
4. 信号衰落

但这更接近真实应用场景！

## 优化建议

1. **固定天线位置**: 使用支架固定天线，避免移动
2. **屏蔽环境**: 在金属盒或屏蔽室中测试
3. **多次测试**: 增加测试次数（50-100次）取平均
4. **记录环境**: 记录测试时的环境条件
